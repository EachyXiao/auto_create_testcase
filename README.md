# Auto Create Testcase (AI 測試案例生成系統)

這是一個基於 AI (Gemini) 的自動化測試案例生成系統。透過提供心智圖或流程圖等原始需求文件，AI 能夠自動分析業務邏輯、識別風險，並產出結構化的測試案例，最終匯出為可直接匯入測試管理工具的 CSV 格式。

## 系統架構 (Vault 結構)

本系統採用類似 Obsidian Vault 的資料夾結構來管理檔案：

```text
├── AI testcase/
│   ├── raw/      # 收件匣：放置原始素材（如心智圖、流程圖的 Markdown 描述），AI 只讀不改。
│   ├── wiki/     # AI 的工作區：存放分析文件與格式化的測試案例。
│   │   ├── log.md      # 系統操作日誌 (append-only)
│   │   └── <topic>/    # 每個功能主題一個專屬資料夾
│   └── output/   # 匯出區：存放最終生成的 CSV 檔案或測試報告。
├── 主次項目定義.md # 測試原則與情境定義（AI 分析的最高指導原則）
├── TC範例參考.csv  # 測試案例的標準格式範本
└── GEMINI.md     # AI 系統提示詞與工作流定義
```

## 工作流程 (Workflow)

本系統定義了三個主要的工作階段 (Skills)，請依序呼叫 AI 執行：

### 1. 分析 (Analyze)
**觸發詞：** 「分析」、「analyze」、「開始分析」

*   **操作方式：** 將需求文件（如心智圖、流程圖）放入 `AI testcase/raw/` 資料夾中，然後對 AI 說「開始分析」。
*   **AI 行為：**
    *   讀取 `raw/` 中的新檔案。
    *   參考 `主次項目定義.md` 識別業務風險與邊界值。
    *   在 `wiki/<topic>/` 下產出 `[feature-name]_analysis.md` (測試計畫與情境拆解)。
    *   更新 `log.md`。

### 2. 撰寫 (Draft)
**觸發詞：** 「撰寫」、「draft」、「根據分析撰寫」

*   **操作方式：** 在分析完成後，對 AI 說「根據 [某某分析文件] 撰寫測試案例」。
*   **AI 行為：**
    *   讀取指定的 `_analysis.md` 文件。
    *   將每一個測試情境擴充為包含前置條件、步驟、預期結果的完整測試案例。
    *   在 `wiki/<topic>/` 下產出 `[feature-name]_test-cases.md`。
    *   更新 `log.md`。

### 3. 匯出 (Export)
**觸發詞：** 「匯出」、「export」、「產出 csv」

*   **操作方式：** 在撰寫完成後，對 AI 說「將 [某某測試案例] 匯出為 CSV」。
*   **AI 行為：**
    *   讀取指定的 `_test-cases.md` 文件。
    *   使用 (或生成) Python 腳本 (`md_to_csv.py`) 將 Markdown 格式轉換為符合 `TC範例參考.csv` 格式的 CSV 檔案。
    *   將結果存放在 `AI testcase/output/` 資料夾中。
    *   更新 `log.md`。

## 核心原則

*   **風險導向：** 系統在分析階段會優先考量資源轉移點、系統邊界與中斷後果。
*   **完整性：** 強制要求覆蓋成功、失敗、異常、中斷、語系、相容性、網路等多種情境。
*   **結構化輸出：** 所有產出的文件皆遵循嚴格的 Markdown 格式，以利後續的自動化解析與轉換。

## 快速開始

1.  將您的需求文件 (Markdown 格式) 放入 `AI testcase/raw/`。
2.  啟動 Gemini CLI 並載入此專案。
3.  輸入指令：`開始分析`。
4.  等待分析完成後，輸入指令：`撰寫測試案例`。
5.  等待撰寫完成後，輸入指令：`匯出 csv`。
6.  前往 `AI testcase/output/` 取得您的測試案例 CSV 檔案。
