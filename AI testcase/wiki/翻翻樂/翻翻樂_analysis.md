# 翻翻樂 - 遊戲主流程 測試分析

## 1. 功能模組
- 翻翻樂 (Flip-Flip Fun)

## 2. 功能點
- 遊戲進入與門票檢查
- 遊戲進行階段 (記憶、準備、進行)
- 遊戲結算與獎勵發放 (Webview 與主程式交互)

## 3. 測試情境拆解

### 3.1 成功結果流程 (Success Outcome Flow)
- **[成功] 遊戲獲勝並成功領取獎勵：** 門票充足 -> 進入遊戲 -> 完成配對 -> Webview 回傳成功 -> 主程式扣票 -> 發放獎勵。
- **[成功] 遊戲失敗但流程完整：** 門票充足 -> 進入遊戲 -> 時間結束未完成配對 -> Webview 回傳失敗 -> 主程式扣票 -> 無獎勵 -> 回到開始頁面。
- **[成功] 點擊「Game Start」進入遊戲：** 驗證門票扣除前的進入邏輯。
- **[成功] 遊戲中配對成功動畫：** 驗證配對成功時的 1 秒鎖定與動畫表現。
- **[成功] 遊戲結束手動點擊回到開始頁面：** 驗證結算動畫後的交互。

### 3.2 失敗結果流程 (Failure Outcome Flow)
- **[失敗] 門票不足無法開始：** 門票數為 0 時，「Game Start」按鈕反灰且不可點擊。
- **[失敗] 門票剛好不足：** 擁有 0 張門票時的介面狀態。
- **[失敗] 遊戲中配對失敗動畫：** 驗證配對失敗時的 1 秒鎖定與牌面覆蓋。

### 3.3 異常處理流程 (Error Handling Flow)
- **[異常] Webview 回傳資料格式錯誤：** 模擬 Webview 回傳非法 JSON 或篡改後的結果，驗證主程式不扣票並回到開始頁面。
- **[異常] 系統記憶體不足：** 在 Webview 遊戲進行中模擬低記憶體狀態，驗證遊戲是否崩潰或能正確恢復。
- **[異常] 門票數據同步異常：** 主程式檢查有票但扣票時伺服器回傳餘額不足。

### 3.4 中斷流程 (Interruption Flow)
- **[中斷] 遊戲進行中 App 進入背景：** 驗證計時器是否暫停或繼續，回到前台後的狀態。
- **[中斷] 遊戲進行中接聽電話：** 驗證 Webview 狀態保持。
- **[中斷] 結算動畫中強制關閉 App：** 驗證下次啟動時門票是否已扣除及獎勵是否補發（風險點）。
- **[中斷] 數據回傳時強制關閉 App：** 驗證主程式與 Webview 的狀態一致性。

### 3.5 語系驗證情境 (Language Verification Scenario)
- **[語系] 繁體中文介面驗證：** 檢查開始頁面、遊戲中提示、結算頁面文案。
- **[語系] 英文介面驗證：** 檢查「Game Start」、倒數計時、獎勵動畫文案。
- **[語系] 日文介面驗證：** 檢查所有 UI 元素在日文下的排版與截斷。

### 3.6 相容性驗證情境 (Compatibility Verification Scenario)
- **[相容性] Android 主流手機 (1080p) 佈局驗證：** 檢查牌面排列、進度條顯示。
- **[相容性] iOS 高解析度手機 (iPhone Pro Max) 佈局驗證：** 檢查紅色脈衝邊框與動畫流暢度。
- **[相容性] 平板裝置 (iPad/Android Tablet) 佈局驗證：** 檢查 Webview 縮放是否正常。

### 3.7 網路異常驗證情境 (Network Anomaly Verification Scenario)
- **[網路] 數據回傳時斷線：** Webview 回傳結果時網路中斷，驗證主程式的重試機制或錯誤處理。
- **[網路] 遊戲開始時弱網：** 點擊 Game Start 後載入 Webview 緩慢，驗證是否有 Loading 提示。
- **[網路] 遊戲進行中網路切換 (Wi-Fi 轉 4G)：** 驗證 Webview 穩定性。

### 3.8 邊界值分析 (Boundary Value Analysis)
- **[邊界] 記憶階段最後 1 秒：** 驗證 5 秒倒數結束瞬間的狀態切換。
- **[邊界] 遊戲進行最後 5 秒：** 驗證紅色脈衝邊框是否準時觸發。
- **[邊界] 遊戲時間剛好在 0 秒完成配對：** 驗證勝負判定臨界點。
- **[邊界] 門票數量上限：** 驗證門票顯示位數（如 99+）。
